# 14. 운영 원칙 (Operation Principles)

기억보다 기록을 우선한다.

중요한 정보는 모두가 공유한다.

직원은 필요한 정보만 본다.

관리자는 모든 데이터를 본다.

AI는 결정을 내리지 않는다.

AI는 근거와 추천만 제공한다.

반복 업무는 자동화한다.

모든 중요한 행동은 기록한다.

누가

언제

무엇을 했는지 남긴다.

화면은 10초 안에 이해되어야 한다.

데이터는 운영의 자산이다.

시간이 지날수록 시스템은 더 똑똑해진다.

좋은 운영은 좋은 기록에서 시작된다.

기록은 기억을 지배한다.

# 16. Development Rules

Specification에 없는 기능은 개발하지 않는다.

새로운 기능은 운영 원칙을 우선한다.

기능보다 사용성을 우선한다.

중복 기능은 만들지 않는다.

기존 데이터를 최대한 활용한다.

직원이 10초 안에 이해하지 못하는 화면은 다시 설계한다.

AI는 충분한 데이터가 쌓인 후 적용한다.

모든 기능은 실제 라이브바 운영을 기준으로 설계한다.

Specification은 확정된 내용만 기록한다.

아이디어는 ideas.md에서 관리한다.

로드맵은 roadmap.md에서 관리한다.

변경 사항은 changelog.md에 기록한다.

개발 과정은 development.md에 기록한다.
